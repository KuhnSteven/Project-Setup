Planning

When creating a new project, it's important to establish what the important needs are: tools, design, and client-needs. Each requires specific solutions and countless articles have been written on the topic of each subject.

Before worrying about how a website will look coding-wise, it is usually best to create 'wireframes' with a program like Adobe XD. This allows for more active planning of the design, like how large certain components should be or color combinations.

Useful steps for project management:

  • Trello Board for tracking activity
  • ReadMe or Documentation for requirements
  • Plan for: Marketing, Technology, Colors & Fonts, etc
  • Structure html for accessibility, design for responsiveness

Github

Github Cheat Sheet

Github Repository Setup Steps:

  • On either on Github.com or the application, create a new repository.
  • Clone the new repo to the desktop working environment.
  • Open a new terminal, either from folder (shortcut ?) or Developer Environment (Ctrl + Cmd + T for Steve's Visual Studio Code)
  • To setup a basic node project, npm init -y , -y automates through the setup questions.
  • Alternatively, npx create-react-app PROJECT-NAME spins up a new base react-app, with the 'PROJECT-NAME' as the name of the folder and thereby project.

Node

React Getting Started Guide

Suggested project dependencies

  • React project: npm i react-redux react-router-prop-types react-scripts
  • Redux project: npm i proptypes redux redux-logger redux-thunk
  • Project design: npm i bootstrap reactstrap react-icons
  • Multi-page project: npm i express axios
  • Users project: npm i uuid jsonwebtoken jwtdecode
  • When ready to test code, npm start from within the main project folder. If project utilizes multiple servers, new scripts to run them concurrently will be necessary.

HTML Setup

Paragraph

basic parts of an html file

  • Head - file detail location, includes title and meta tags, not inlcuded with a
  • Header - Top of the page typically, may contain navigation-links, text, logo, slogan, and more.
    • Image - logo, background, account icon, icons, etc
    • Nav - holder for links
  • Main - container for overall content
    • Body - majority of the content, typically the primary focus
      • ul / ol & li - (un-)ordered list, and list-item which represents each item in a list
      • link (a) - can reference an internal (src) or external (href) link
    • div - container, often the primary causality of div-itis.
    • Section - portion of content that holds smaller content
      • Article - smaller portion, typically holds text and images
      • Card - smallest(?) portion
      • Paragraph - text field
  • Footer - Bottom of the page (probably, I'm not your boss)
